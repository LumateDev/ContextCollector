# Context Collector

Утилита для сбора контекста проекта в текстовый файл или Markdown с фильтрацией файлов и поддержкой `.gitignore`.

![Python](https://img.shields.io/badge/python-3.8+-green)
![License](https://img.shields.io/badge/license-MIT-lightgrey)

## 📌 Описание

**Context Collector** — это настольное приложение с графическим интерфейсом на базе CustomTkinter. Оно сканирует выбранную папку проекта, собирает содержимое текстовых файлов (с учётом расширений, исключённых папок и правил `.gitignore`) и формирует единый файл-отчёт в формате **TXT** или **Markdown**. Результат можно сохранить или сразу скопировать в буфер обмена.

Идеально подходит для:

- передачи контекста проекта в AI-ассистенты (ChatGPT, Copilot и др.);
- быстрого создания документации по коду;
- подготовки отчётов для ревью.

## ✨ Особенности

- 🖥️ **Простой GUI** — выбирайте папку, настраивайте фильтры, запускайте сбор.
- 📁 **Умная фильтрация** — включайте файлы по расширениям, исключайте ненужные папки.
- 🔒 **Поддержка `.gitignore`** — автоматически учитывает правила игнорирования из корня проекта.
- 📊 **Дерево проекта** — опционально добавляет в отчёт структуру папок.
- 📄 **Два формата вывода** — обычный текст (TXT) или валидный Markdown с подсветкой синтаксиса.
- 🛠️ **Настраиваемые фильтры** — расширения и исключённые папки редактируются через встроенный диалог.
- 🧹 **Обработка бинарных файлов** — пропускает бинарные данные, предупреждая о них.
- 💾 **Сохранение настроек** — все параметры сохраняются в `%APPDATA%/ContextCollector/` (Windows) или аналогичной папке на macOS/Linux.

## 🚀 Установка и запуск

### Вариант 1: Запуск из исходного кода

1. **Клонируйте репозиторий:**

   ```bash
   git clone https://github.com/ваш_пользователь/ContextCollector.git
   cd ContextCollector
   ```

2. **Установите зависимости:**

   ```bash
   pip install -r requirements.txt
   ```

3. **Запустите приложение:**
   ```bash
   python main.py
   ```

### Вариант 2: Сборка исполняемого файла (Windows)

Просто запустите `build.bat`. Скрипт:

- создаст виртуальное окружение;
- установит зависимости;
- соберёт `ContextCollector.exe` с помощью PyInstaller.

Готовый `.exe` появится в папке `dist/`.

## 🖱️ Использование

1. **Выберите папку проекта** — нажмите «Обзор…» или вставьте путь вручную.
2. **Настройте параметры**:
   - **Формат** — TXT или Markdown.
   - **Макс. размер файла (KB)** — файлы больше этого размера будут пропущены.
   - **Учитывать .gitignore** — включает/выключает поддержку правил из `.gitignore`.
   - **Дерево файлов** — добавить в отчёт древовидную структуру папок.
   - **Копировать в буфер** — автоматически скопировать результат после завершения.
3. **Фильтры** — нажмите «Фильтры», чтобы изменить список расширений и исключённых папок.
4. **Нажмите «Собрать»**. Процесс сканирования отобразится на прогресс-баре и в строке статуса.
5. После завершения появится диалог сохранения. Выберите имя и место для файла отчёта.
   - Если включено копирование в буфер, результат будет доступен для вставки сразу.
   - Если сохранение отменено, программа предложит скопировать текст в буфер.

## ⚙️ Настройка фильтров

В диалоге «Настройка фильтров» вы можете:

- **Расширения файлов** — каждое расширение с новой строки. Можно указывать с точкой (`.py`) или без (`py`). Поддерживаются любые текстовые форматы.
- **Исключённые папки** — имена папок, которые будут пропущены при обходе (например, `node_modules`, `.git`).

Кнопка «Сбросить к стандартным» возвращает предопределённые списки (см. `config.py`).

## 📁 Структура проекта

```
ContextCollector/
├── .gitignore            # Игнорируемые файлы для Git
├── build.bat             # Скрипт сборки .exe (Windows)
├── config.py             # Настройки, пути, константы
├── formatter.py          # Формирование TXT/Markdown отчёта
├── main.py               # Главное окно и логика GUI
├── requirements.txt      # Зависимости Python
└── scanner.py            # Сканер файлов, фильтрация, .gitignore
```
